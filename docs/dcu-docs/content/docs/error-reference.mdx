title: Error Reference
description: All error types returned by the DCU SDK and how to handle them.

Every SDK endpoint returns `Effect.Effect<TxSignBuilder, DcuError, never>`. When using `safeRun()`, the `Either.Left` value is always a `DcuError`, a tagged union you can match against precisely.

## Discriminating Errors

Each error has a `_tag` field for exhaustive matching:

```ts
import { Either } from "effect";
import { createAccount } from "@tx-meta/dcu-kit";

const result = await createAccount(lucid, config).safeRun();

if (Either.isLeft(result)) {
  const error = result.left;

  switch (error._tag) {
    case "TransactionBuildError":
      console.error(`Build failed in ${error.operation}:`, error.error);
      break;
    case "UtxoNotFound":
      console.error(`UTxO not found: ${error.tokenName} at ${error.address}`);
      break;
    case "InvalidDatum":
      console.error(`Invalid datum field "${error.field}":`, error.reason);
      break;
    default:
      console.error("Unexpected error:", error);
  }
}
```

## Error Types

### `TransactionBuildError`
The most common error. Raised when the SDK fails to construct a valid transaction.

| Field | Type | Description |
|-------|------|-------------|
| `operation` | `string` | The endpoint name where the failure occurred (e.g. `"distributePayout"`) |
| `error` | `string` | The underlying error message from Lucid or the SDK |

```ts
if (error._tag === "TransactionBuildError") {
  console.error(`[${error.operation}] ${error.error}`);
}
```

---

### `UtxoNotFoundError`
Raised when a required UTxO or token cannot be located at the expected address.

| Field | Type | Description |
|-------|------|-------------|
| `tokenName` | `string` | The token name being searched for |
| `address` | `string` | The address where it was expected |

---

### `InvalidDatumError`
Raised when a UTxO datum fails a runtime validation check (e.g. wrong variant, unexpected state).

| Field | Type | Description |
|-------|------|-------------|
| `field` | `string` | The datum field or variant that was invalid |
| `reason` | `string` | Human-readable explanation |

---

### `DatumDecodingError`
Raised when a raw CBOR datum cannot be decoded against the expected schema.

| Field | Type | Description |
|-------|------|-------------|
| `utxoId` | `string` | The UTxO identifier that failed to decode |
| `error` | `string` | The decoding error detail |

---

### `InsufficientUtxosError`
Raised when the wallet does not have enough UTxOs to satisfy the transaction requirements.

| Field | Type | Description |
|-------|------|-------------|
| `required` | `number` | Number of UTxOs required |
| `available` | `number` | Number of UTxOs available |

---

### `LucidError`
A generic wrapper around errors thrown by the Lucid Evolution library itself.

| Field | Type | Description |
|-------|------|-------------|
| `message` | `string` | The Lucid error message |

---

### Rarely Encountered

These errors indicate misconfiguration rather than user input errors:

| Error | `_tag` | When it occurs |
|-------|--------|----------------|
| `TransactionSignError` | `"TransactionSignError"` | Wallet signing failed |
| `TransactionSubmitError` | `"TransactionSubmitError"` | Chain submission rejected |
| `ValidatorNotFoundError` | `"ValidatorNotFound"` | A compiled validator is missing from the blueprint |
| `BlueprintLoadError` | `"BlueprintLoadError"` | The Aiken-compiled blueprint JSON could not be loaded |
| `ConfigurationError` | `"ConfigurationError"` | A required SDK configuration key is absent |
| `SetupError` | `"SetupError"` | General SDK initialisation failure |

---

## Full `DcuError` Union

```ts
import type { DcuError } from "@tx-meta/dcu-kit";

// DcuError is:
type DcuError =
  | TransactionBuildError
  | UtxoNotFoundError
  | InvalidDatumError
  | DatumDecodingError
  | InsufficientUtxosError
  | LucidError
  | TransactionSignError
  | TransactionSubmitError
  | ValidatorNotFoundError
  | BlueprintLoadError
  | ConfigurationError
  | SetupError;
```
