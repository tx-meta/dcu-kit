The **DCU Toolkit** acts as the crucial middleware connecting off-chain application logic to the on-chain Aiken smart contracts.

## High Level Overview

## Contract Ecosystem

The SDK relies on three principal Plutus V3 validators:

### 1. Account Validator
Provides global identification for a user. It stores basic encrypted metadata (like hashed email and phone) inside the Reference NFT datum.

### 2. Group Validator
Defines the parameters of a specific DCU instance, such as:
* Admin authorization (via `GroupAdmin` NFT)
* Minimum contribution limits
* The length of time between payout rotation cycles.

### 3. Treasury Validator
Holds the actual pooled funds. Users lock their ADA into the Treasury, where it sits securely until the `distributePayout` endpoint rotates the payout to the current interval's eligible member.
