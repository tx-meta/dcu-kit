---
title: deleteGroup
description: Burn an empty, deactivated group and reclaim its locked ADA.
---

The `deleteGroup` endpoint permanently tears down a group: it burns both group tokens and returns the locked ADA (including the `creator_bond`) to the admin.

<Callout title="Bound endpoint" type="info">
Obtained from `createDcuSdk(settingsPolicy)`. See [Getting Started](/docs/getting-started).
</Callout>

### Constraints
- The group must already be **deactivated** (`is_active == false`) — set this first via [`updateGroup`](/docs/api-reference/updateGroup).
- `member_count` must be `0` — every member must have exited.

### What it does
- Burns both group tokens (Reference 100 + Admin Auth 222).
- Returns all ADA locked in the group UTxO, including the `creator_bond`, to the admin as change.

## Configuration

```ts
export type DeleteGroupConfig = {
  /** 28-byte token suffix of the (deactivated, empty) group. */
  groupTokenSuffix: string;
};
```

## Usage

```ts
import { createDcuSdk } from "@tx-meta/dcu-sdk";

const sdk = createDcuSdk(settingsPolicy);

try {
  const txBuilder = await sdk.deleteGroup(lucid, { groupTokenSuffix }).unsafeRun();
  const txHash = await txBuilder.sign.withWallet().complete().then((tx) => tx.submit());
  console.log("Group deleted! TxHash:", txHash);
} catch (error) {
  console.error("Failed to delete group:", error);
}
```
