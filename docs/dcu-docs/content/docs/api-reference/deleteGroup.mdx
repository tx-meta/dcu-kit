---
title: deleteGroup
description: Deactivate a DCU Group and prevent any future memberships.
---

The `deleteGroup` endpoint performs a programmatic "soft delete" of a Decentralized Credit Union. 

## Protocol Constraints
You **cannot** completely destroy the Group Reference UTxO natively if it holds critical historical data. Instead, this endpoint updates the Group's datum state to `is_active: false`. Furthermore, the smart contract strictly validates that the group's `member_count` evaluates to absolutely `0` before allowing deactivation.

## Configuration

```ts
import { UTxO } from "@lucid-evolution/lucid";
import { GroupDatum } from "@dcu/sdk";

export type DeleteGroupConfig = {
    /** The Group's master state UTxO */
    groupUtxo: UTxO;
    
    /** The currently active group parameters */
    currentDatum: GroupDatum;
    
    /** The Administrator's Auth UTxO proving possession of the Admin credentials */
    adminUtxo: UTxO;
};
```

## Obtaining `currentDatum`

`currentDatum` must be decoded from the inline datum on the `groupUtxo`. Use `Data.from` with the `GroupDatum` schema:

```ts
import { Data } from "@lucid-evolution/lucid";
import { GroupDatum } from "@dcu/sdk";

if (!groupUtxo.datum) throw new Error("Group UTxO has no inline datum");
const currentDatum = Data.from(groupUtxo.datum, GroupDatum);
```

## Internal Execution Mechanics
When an admin triggers this transaction:
1. It validates the `adminUtxo` as the holder of the `GroupAdmin` NFT.
2. It copies `currentDatum` and sets `is_active` to `false`.
3. It pushes the deactivated datum back to the `GroupValidator`.

## Usage

```ts
import { deleteGroup } from "@dcu/sdk";

try {
    const txBuilder = await deleteGroup(lucid, config).unsafeRun();
    const txHash = await txBuilder.sign.withWallet().complete().then(tx => tx.submit());
    console.log("Group successfully deactivated! TxHash:", txHash);
} catch (error) {
    console.error("Failed to deactivate group:", error);
}
```
