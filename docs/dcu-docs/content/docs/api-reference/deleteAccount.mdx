The `deleteAccount` endpoint allows a user to completely remove their identity from the DCU protocol. It guarantees that the user leaves no dangling reference metadata behind.

## Security Constraints
You **cannot** delete an account if that account is currently an active member of any DCU Group. The `deleteAccount` transaction sweeps the Treasury validators looking for your specific Reference Token. If a match is found, the SDK yields a `TransactionBuildError` protecting the integrity of the protocol.

## Configuration

```ts
import { UTxO } from "@lucid-evolution/lucid";

export type DeleteAccountConfig = {
    /** The UTxO in your wallet holding the User Auth NFT */
    user_utxo: UTxO;
    /** The UTxO sitting at the Account Validator holding the Reference NFT */
    account_utxo: UTxO;
};
```

## Internal Execution Mechanics
When you trigger this endpoint, the SDK:
1. Uses `findCip68TokenPair` to match the exact User Token and Reference Token.
2. Performs a parallel `Effect.all` search against every known Treasury UTxO to ensure this token is **not** an active member of any pod.
3. Constructs a transaction that simultaneously **burns** both tokens and redeems the min-ADA locked in the Reference NFT back to the user.

## Usage

```ts
import { deleteAccount } from "@dcu/sdk";

try {
    const txBuilder = await deleteAccount(lucid, config).unsafeRun();
    const txHash = await txBuilder.sign.withWallet().complete().then(tx => tx.submit());
    console.log("Account successfully deleted! TxHash:", txHash);
} catch (error) {
    console.error("Failed to delete account:", error);
}
```
