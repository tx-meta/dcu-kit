title: deleteAccount

The `deleteAccount` endpoint removes a user's identity, burning both CIP-68 account tokens and reclaiming the locked min-ADA.

<Callout title="Bound endpoint" type="info">
`deleteAccount` is obtained from `createDcuSdk(settingsPolicy)` — it scans the deployment's treasuries for active memberships before burning, so it needs the protocol context.
</Callout>

### Integrity check
You **cannot** delete an account that is still an active member of any group. The transaction sweeps the deployment's treasury UTxOs for this account's token; if a membership is found, the SDK yields a `TransactionBuildError`, protecting protocol integrity.

### What it does
- Matches the exact User Auth (222) and Reference (100) token pair.
- Verifies the account holds no active membership across the deployment's treasuries.
- Burns both tokens and returns the reference NFT's min-ADA to the caller.

## Configuration

```ts
export type DeleteAccountConfig = {
  /** 28-byte token suffix of the account to delete. */
  accountTokenSuffix: string;
};
```

## Usage

```ts
import { createDcuSdk } from "@tx-meta/dcu-kit";

const sdk = createDcuSdk(settingsPolicy);

try {
  const txBuilder = await sdk.deleteAccount(lucid, { accountTokenSuffix }).unsafeRun();
  const txHash = await txBuilder.sign.withWallet().complete().then((tx) => tx.submit());
  console.log("Account deleted! TxHash:", txHash);
} catch (error) {
  console.error("Failed to delete account:", error);
}
```
