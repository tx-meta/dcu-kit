title: updateGroup

The `updateGroup` endpoint lets a group admin reconfigure their pod, including deactivating it (`is_active = false`) ahead of deletion.

<Callout title="Bound endpoint" type="info">
Obtained from `createDcuSdk(settingsPolicy)`. See [Getting Started](/docs/getting-started).
</Callout>

### Security constraints
Critical financial parameters (fees, `interval_length`, collateral/payout policy) are sensitive: the Group Validator rejects any transaction mutating them unless the group is empty (`member_count == 0`). Authorization requires the group admin (222) token.

### What it does
- Spends the group UTxO and re-locks it with `updatedDatum`.
- Permits critical-field changes only while `member_count == 0`.
- Commonly used to set `is_active = false` before [`deleteGroup`](/docs/api-reference/deleteGroup).

## Configuration

```ts
import { GroupDatum } from "@tx-meta/dcu-kit";

export type UpdateGroupConfig = {
  /** 28-byte token suffix of the group. */
  groupTokenSuffix: string;

  /** The full desired group datum (see createGroup for the field reference). */
  updatedDatum: GroupDatum;
};
```

## Usage

```ts
import { createDcuSdk } from "@tx-meta/dcu-kit";

const sdk = createDcuSdk(settingsPolicy);

// e.g. deactivate the group ahead of deletion
const updatedDatum = { ...currentDatum, is_active: false };

try {
  const txBuilder = await sdk.updateGroup(lucid, {
    groupTokenSuffix,
    updatedDatum,
  }).unsafeRun();
  const txHash = await txBuilder.sign.withWallet().complete().then((tx) => tx.submit());
  console.log("Group updated! TxHash:", txHash);
} catch (error) {
  console.error("Failed to update group:", error);
}
```
