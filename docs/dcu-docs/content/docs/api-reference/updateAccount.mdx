---
title: updateAccount
description: Update the display name and contact stored in your account.
---

The `updateAccount` endpoint refreshes the `AccountDatum` (`display_name`, `contact`) held at the Account Validator. Authorization requires the matching User Auth NFT in your wallet.

<Callout title="Standalone endpoint" type="info">
`updateAccount` does not depend on a deployment's settings, so it is imported and called directly (not via `createDcuSdk`).
</Callout>

### What it does
- Updates `display_name` and `contact` (raw UTF-8) on the account's reference datum; both default to the wallet address when omitted.
- Requires the User Auth NFT in the wallet for authorization (the contract validates the CIP-68 pair).
- Returns the auth token to your wallet and re-locks the updated datum at the validator.

## Configuration

```ts
export type UpdateAccountConfig = {
  /** 28-byte token suffix of the account to update. */
  accountTokenSuffix: string;

  /** Optional new UTF-8 display name. Defaults to wallet address when omitted. */
  display_name?: string;

  /** Optional new UTF-8 secondary contact. Defaults to wallet address when omitted. */
  contact?: string;
};
```

## Usage

```ts
import { updateAccount } from "@tx-meta/dcu-sdk";

try {
  const txBuilder = await updateAccount(lucid, {
    accountTokenSuffix,
    display_name: "alice.ada",
    contact: "alice@example.com",
  }).unsafeRun();

  const txHash = await txBuilder.sign.withWallet().complete().then((tx) => tx.submit());
  console.log("Account updated! TxHash:", txHash);
} catch (error) {
  console.error("Failed to update account:", error);
}
```
