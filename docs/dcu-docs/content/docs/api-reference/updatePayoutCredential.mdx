The `updatePayoutCredential` endpoint updates the `member_payment_credential` stored in a member's treasury datum, so future **Push**-mode payouts land at the member's current wallet. Use it after rotating receiving wallets.

<Callout title="Bound endpoint" type="info">
Obtained from `createDcuSdk(settingsPolicy)`. See [Getting Started](/docs/getting-started).
</Callout>

### What it does
- Updates `member_payment_credential` in the treasury UTxO datum.
- The new credential is derived from the signing wallet's address (the account token input's payment key hash), proving wallet control — the on-chain validator re-derives and verifies it.
- All other datum fields are preserved.
- Requires the member's account NFT in the wallet for authorization.

<Callout title="Push vs Pull" type="info">
This affects **Push** mode, where the pot is paid to a stored credential. In **Pull** mode the payout is claimed by token possession via [`claimPayout`](/docs/api-reference/claimPayout), so the stored credential is not the gating factor.
</Callout>

## Configuration

```ts
export type UpdatePayoutCredentialConfig = {
  /** 28-byte token suffix of the caller's account. */
  accountTokenSuffix: string;
};
```

## Usage

```ts
import { createDcuSdk } from "@tx-meta/dcu-sdk";

const sdk = createDcuSdk(settingsPolicy);

const txBuilder = await sdk.updatePayoutCredential(lucid, { accountTokenSuffix }).unsafeRun();
const txHash = await txBuilder.sign.withWallet().complete().then((tx) => tx.submit());
console.log("Payout credential updated:", txHash);
```
